# Shared JSON Schemas

All workflow outputs write to `data/snapshots/`. All renderers read from `data/snapshots/`. This is the contract.

```json
      "value_usd": 174300000000,
```

```json
  "gurus": [
    {
      "id": "berkshire_hathaway",
      "cik": "0001067983",
      "display": "Berkshire Hathaway"
    },
    { "id": "michael_burry", "cik": "0001649339", "display": "Michael Burry" },
    {
      "id": "bill_ackman",
      "cik": "0001336528",
      "display": "Bill Ackman / Pershing"
    }
  ],
  "insider_watch": ["TSLA", "NVDA", "META"]

```

## Rules

- All dates in ISO 8601 (`YYYY-MM-DD`)
- All quarters in `YYYY[Q1|Q2|Q3|Q4]` format
- All money values in USD, no formatting (integers preferred)
- All Hebrew strings UTF-8, RTL-safe (no embedded LTR English without bidi marks if needed for rendering)
- File names lowercase, snake_case for guru IDs
- Atomic writes (write to `.tmp` then rename)

## 8. `alerts-{date}.json` - Derived alerts (latest wins)

Generated by `scripts/derive-alerts.py`. Rolling: one file per UTC day, the most recent overwrites and is consumed by the dashboard.

```json
{
  "generated_at": "2026-05-17T09:00:00Z",
  "alerts": [
    {
      "tag": "INSIDER BUY",
      "severity": "info|success|warn|danger",
      "text_he": "מנכל TSLA קנה $4.2M ב-2026-05-15",
      "ticker": "TSLA",
      "value_usd": 4200000,
      "source_file": "data/snapshots/insider-TSLA.json"
    }
  ]
}
```

Alert tags + rules (thresholds are constants at the top of `scripts/derive-alerts.py`):

| tag                       | severity | trigger                                                             |
| ------------------------- | -------- | ------------------------------------------------------------------- |
| `INSIDER BUY`             | success  | any insider trade with `action=="buy"` and `value_usd >= 1_000_000` |
| `INSIDER SELL CLUSTER`    | warn     | same ticker has >= 3 sells in 14 days totalling >= $5M              |
| `NEW GURU POSITION`       | info     | `diff-*.json` `new[]` entry with `value_usd >= 100_000_000`         |
| `GURU EXIT`               | warn     | `diff-*.json` `exited[]` entry with `value_usd >= 50_000_000`       |
| `LARGE POSITION INCREASE` | info     | `diff-*.json` `increased[]` entry with `delta_pct >= 50`            |
| `STALE DATA`              | warn     | `holdings-*.json` whose `filing_date` is > 90 days old              |

Notes:

- `ticker` may be `null` for alerts that aren't ticker-scoped (e.g. `STALE DATA`).
- `source_file` is repo-relative for traceability.
- `value_usd` may be `0` when not applicable (e.g. `LARGE POSITION INCREASE` carries only `delta_pct`).
